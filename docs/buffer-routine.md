# SiamAthlete buffer-filler routine — สเปกอ้างอิง (สำหรับ Cowork scheduled task)

Task นี้รันทุก 3 ชั่วโมง หน้าที่คือรักษาคิวโพสต์ล่วงหน้าให้เต็ม **5 วัน** เขียนเติมเฉพาะเมื่อคิวเหลือ **< 3 วัน** รอบไหนคิวเต็มให้จบทันที (ประหยัดโควตา)

## Secret
- อ่านจากไฟล์ `C:\Users\Jongrak HO\content-console\.env.local`
- คีย์ = **`CRON_SECRET`** (บรรทัด `CRON_SECRET=...`)
- ใช้เป็น HTTP header ชื่อ **`x-cron-secret`** ในทุก request · อย่าพิมพ์ค่าออกมา
- (ค่าเดียวกันตั้งไว้ใน Vercel env แล้ว ทั้ง server-side cron และ routine ใช้ตัวนี้ร่วมกัน)

Base URL = `https://content-console-indol.vercel.app`

## 1) เช็กคิว — `GET /api/schedule/plan`
Header: `x-cron-secret`
Response:
```json
{ "postsPerDay": 4, "targetDays": 5, "minDays": 3,
  "scheduledFuture": 8, "daysCovered": 2.0, "needRefill": true,
  "openSlots": [ { "scheduled_at": "2026-07-14T03:00:00.000Z", "bkk_date": "2026-07-14", "bkk_time": "10:00", "pillar": "A" } ] }
```
- `needRefill=false` หรือ `openSlots` ว่าง → คิวพอ **จบงาน** ไม่เขียน
- `needRefill=true` → เขียน 1 โพสต์ต่อ 1 slot ใน `openSlots` ให้ครบ (ไม่มากไม่น้อยกว่านั้น)

## กันเขียนเกิน 5 วัน
`openSlots` ถูกจำกัดไม่เกิน `targetDays` (5 วัน) อยู่แล้ว → เขียนเฉพาะช่องที่ API คืนมา **ห้ามคิด scheduled_at เอง / ห้ามสร้างช่องเพิ่ม** ทำแบบนี้คิวจะไม่เกิน 5 วันโดยอัตโนมัติ

## 2) กันซ้ำ — `GET /api/articles`
อ่าน `topic`/`title` ~15–20 ชิ้นล่าสุด เลี่ยงหัวข้อย่อยที่ซ้ำ 7 วันก่อนหน้าและซ้ำกันเองในรอบนี้

## 3) เสาหลักตามเวลา (ใช้ `pillar` ของแต่ละ slot — คำนวณตามเวลาไทยแล้ว)
`A` การฝึก · `B` โภชนาการ/ลดไขมัน · `C` ฮอร์โมน&สรีรวิทยา · `D` เตรียมแข่ง · `E` ประวัติ AAS (เชิงการศึกษา) · `F` หัวข้อเชิงลึก
(map เวลา: ~10:00→A · ~16:00→B · ~19:00→C วันคู่/D วันคี่ · ~21:00→E วันคู่/F วันคี่)

## 4) กติกาเนื้อหา
- 1500–2000 ตัวอักษร (นับช่องว่าง) · เจาะหัวข้อเดียวให้ลึกและจบ
- **ห้ามอักขระพิเศษ `-` `:` `,` ในตัว body** (ช่วงตัวเลขใช้คำว่า "ถึง") · ขึ้นบรรทัดใหม่ + แฮชแท็ก `#` ได้
- สไตล์คนโพสจริง คุยกับเพื่อนเล่นเวท มีจุดยืน ฟันธง ทุบความเชื่อผิด · ห้ามภาษาปั่นแบบ AI
- อ้างงานวิจัยแบบเนียน ห้ามมั่วตัวเลข ไม่ชัวร์พูดกว้างอย่างซื่อสัตย์
- ปิดท้าย takeaway + `#SiamAthlete #สร้างกล้ามเนื้อ #เพาะกาย #ลดไขมัน #ความรู้ฟิตเนส`

### Safety
- AAS/PED: เฉพาะประวัติ + กลไก + ความเสี่ยงตามงานวิจัย · **ห้าม** dose/cycle/stack/แหล่งซื้อ/วิธีหลบตรวจ
- คัต/peak week: เชิงความรู้ + เตือนความเสี่ยง เน้นยั่งยืนปลอดภัย

## 5) references
2–4 แหล่งจริง (ผู้เขียน + ปี + วารสาร/ชื่อ study) **ห้ามแต่งลิงก์ปลอม** — ส่งใน field `refs` เท่านั้น **ห้ามใส่ใน body** (ระบบจะโพสต์เป็นคอมเมนต์ใต้โพสต์ให้เอง)

## 6) ส่งเข้าระบบ — `POST /api/ingest` (1 POST ต่อ 1 slot)
Headers: `x-cron-secret`, `content-type: application/json`
```json
{
  "topic": "<เสาหลัก + หัวข้อย่อย>",
  "title": "<หัวเรื่อง ≤70 ตัวอักษร>",
  "excerpt": "<takeaway 1 บรรทัด>",
  "body": "<เนื้อโพสต์เต็ม + แฮชแท็ก (ห้ามมี refs)>",
  "refs": ["<อ้างอิง 1>", "<อ้างอิง 2>"],
  "scheduled_at": "<openSlots[i].scheduled_at เป๊ะ>"
}
```
- `scheduled_at` = ค่าของ slot นั้นเป๊ะ ๆ (จับคู่ `pillar` ↔ slot ให้ถูก)
- **ไม่ต้องส่ง `image_url`** — ระบบเลือกรูปฟรีตามหัวข้อให้เอง
- ต้องได้ `200` + `article.id` (401 = secret ผิด · 400 = ข้อมูลไม่ครบ)

## หลังปล่อย (ระบบทำเอง ไม่เกี่ยวกับ routine)
โพสต์เข้าคิว `scheduled` → Vercel Cron (`/api/publish-due`, ทุก 10 นาที, คลาวด์ 24/7) ปล่อยขึ้นเพจตามเวลา → โพสต์ `refs` เป็นคอมเมนต์ใต้โพสต์ + แจ้ง FYI เข้ากลุ่ม LINE (ไม่มีปุ่ม/ไม่ต้อง approve)
